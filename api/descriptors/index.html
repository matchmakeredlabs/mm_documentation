/descriptors

This endpoint allows users to access descriptors in MatchMaker's database through different query mechanisms.

Table of Contents​

EndpointDescription
GET - By IDRetrieve a descriptor by its unique UUID.
GET - By Element KeyRetrieve descriptors using a composite element key.
GET - By KeywordSearch for descriptors using keyword-based search.
GET - By URLRetrieve descriptors associated with a specific URL.
POST - Add New DescriptorAdd a new descriptor by sending JSON data directly.
POST - Import Descriptor from CSVBulk import descriptors from a CSV file.
PUT - Update Existing DescriptorUpdate an existing descriptor by ID.
DELETEDelete a descriptor by its ID.

GET - By ID​

Endpoint​

https://matchmakeredlabs.net/api/descriptors/{descriptorId}

Retrieve a descriptor by its UUID.

Request​

Request Parameters​

ParameterTypeRequiredDescription

GET - By Element Key​

Endpoint​

GET https://matchmakeredlabs.net/api/descriptors?searchKey={searchKey}&eleType={eleType}

Retrieve a descriptor by element key.

Request​

Request Parameters​

ParameterTypeRequiredDescription
searchKeystringYesAn element key, made up of multiple intermediary statement keys. Ex: "5001C-5073C-6200C-4955P-5247P-5358P-6350P-6736P-13396P"
eleTypestringNoThe type of the educational element. Options include:
→ "any" — Any type
→ "cr" — Curriculum
→ "lr" — Learning Resource
→ "c" — Competency
→ "o" — Other

Sample Requests​

  • cURL
  • JavaScript
  • Python
curl \

  -X GET \
  "https://matchmakeredlabs.net/api/descriptors?searchKey={searchKey}&eleType={eleType}"

async function searchDescriptorByKey(url = "") {

    },
  });
  return response.json();
}

async function getDescriptorByURL(url = "") {

POST - Add New Descriptor​

Endpoint​

https://matchmakeredlabs.net/api/descriptors

Add a new descriptor by sending JSON data directly.

Request​

Request Body​

{
   "eleType": "<eleType>",
   "name": "<name>",
   "url": "<url>",
   "about": "<about>",
   "abstract": "<abstract>",
   "identifier": "<identifier>",
   "educationalLevel": "<educationalLevel>",
   "creator": "<creator>",
   "provenance": "<provenance>",
   "key": "https://matchmakeredlabs.net/api/key/<key1>-<key2>-<key3>"
}

Request Parameters​

ParameterTypeRequiredDescription
eleTypestringYesThe type of the educational element. Options: any, c (curriculum), lr (learning resource), cs (competency statement), o (other).
namestringRecommendedName of the descriptor. Example: Grade 8: The Number System.
urlstringRecommendedThe URL for the educational element.
aboutstringNoInformation about the educational element.
abstractstringRecommendedDescribes the educational element, elaborating on learning outcomes. Example: Know that numbers that are not rational are called irrational....
identifierstringNoThe associated identifier for an educational element. Example: CCSS.MATH.CONTENT.8.NS.A.2.
educationalLevelstringNoThe grade level of the educational element. Example: 8.
creatorstringRecommendedThe creator of the educational element. Example: Purplemath.
provenancestringNoThe entity that described the educational element. Example: David Longdon.
keystringYesThe URL for the associated element key.

Sample Request​

  • cURL
  • JavaScript
  • Python
curl \ 
-H "Authorization: Bearer $TOKEN" \ 
-d '{"eleType":"<eleType>","name":"<name>","url":"<url>","about":"<about>","abstract":"<abstract>","identifier":"<identifier>","educationalLevel":"<educationalLevel>","creator":"<creator>","provenance":"<provenance>","key":"https://matchmakeredlabs.net/api/key/<key1>-<key2>-<key3>"}' \ 
-X POST "https://matchmakeredlabs.net/api/descriptors"
async function postDescriptor(url = "") {
    const response = await fetch(url, {
      method: "POST",
      mode: "cors",
      headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${TOKEN}`
      },
      body: JSON.stringify({
           "eleType": "<eleType>",
           "name": "<name>",
           "url": "<url>",
           "about": "<about>",
           "abstract": "<abstract>",
           "identifier": "<identifier>",
           "educationalLevel": "<educationalLevel>",
           "creator": "<creator>",
           "provenance": "<provenance>",
           "key": "https://matchmakeredlabs.net/api/key/<key1>-<key2>-<key3>"
      })
    });
    return response;
}

POST - Import Descriptor from CSV​

Endpoint​

https://matchmakeredlabs.net/api/descriptors?provenance={provenance}

Request​

Request Body​

In the request, the file with comma-separated values is read with an @, as seen in the cURL request below.

The CSV should contain the following columns:

  • Name
  • ElementType
  • Url
  • About
  • Abstract
  • Identifier
  • MainEntity
  • IsPartOf
  • Position
CSV Column Descriptions​
ColumnTypeDescription
namestringName of the descriptor.
Example: Grade 8: The Number System
eleTypestringElement type options: any, c (curriculum), lr (learning resource), cs (competency statement), o (other)
urlstringThe URL for the educational element.
aboutstringSubject of the educational element.
abstractstringAbstract elaborating on learning outcomes.
Example: Know that numbers that are not rational are called irrational...
identifierstringThe associated identifier (e.g., a standard) for an educational element.
Example: CCSS.MATH.CONTENT.8.NS.A.2
mainEntitystringPrimary entity described in a page or other CreativeWork.
isPartOfstringThe URL of the descriptor's parent element.

Request Parameters​

ParameterTypeRequiredDescription
provenancestringYesThe URI-encoded string of the person or entity that described an educational element.
Example: David Longdon

Example Request​

  • cURL
curl \
-H "Content-Type: text/csv" \
-H "Authorization: Bearer $TOKEN" \
--data-binary @sample.csv \
-X POST "https://matchmakeredlabs.net/api/descriptors?provenance={provenance}"

Response​

Response Parameters​

There are no specific response parameters; a string is returned.

Example Response​

Created descriptor d6d701ef-29d5-4001-9e0a-676ea8ded451, "S"

descriptorIdUUIDYesUUID of descriptor for an educational element.
eleTypestringRecommendedElement type: any, c, lr, cs, o.
namestringRecommendedName of the descriptor.
Example: Grade 8: The Number System
urlstringRecommendedURL for the educational element.
aboutstringRecommendedSubject of the educational element.
abstractstringRecommendedLearning outcome details.
Example: Know that numbers that are not rational are called irrational...
identifierstringRecommendedIdentifier for the element.
Example: CCSS.MATH.CONTENT.8.NS.A.2
educationalLevelstringNoGrade level associated with the element.
Example: 8
creatorstringRecommendedCreator of the element.
Example: Purplemath
provenancestringRecommendedEntity describing the element.
Example: David Longdon
keystringRecommendedURL for the associated element key.

async function updateDescriptor(url = "") {
    const response = await fetch(url, {
      method: "PUT",
      headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${TOKEN}`
      },
      body: JSON.stringify({
           "eleType": "<eleType>",
           "name": "<name>",
           "url": "<url>",
           "about": "<about>",
           "abstract": "<abstract>",
           "identifier": "<identifier>",
           "educationalLevel": "<educationalLevel>",
           "creator": "<creator>",
           "provenance": "<provenance>",
           "key": "https://matchmakeredlabs.net/api/key/<key1>-<key2>-<key3>"
      })
    });
    return response;
}

let TOKEN = "st=Core&un=<username>&x=21234602T21560217Z&m=0qhd9jkTaIqbT_V5xTUmQ4HJhMpd3h5AAB9p9AnceUCs";

async function deleteDescriptorById(url = "") {
    const response = await fetch(url, {
      method: "DELETE",
      mode: "cors",
      headers: {
          'Content-Type': 'application/x-www-form-urlencoded',
          'Authorization': `Bearer ${TOKEN}`
      },
    });
    return response.json();
}
