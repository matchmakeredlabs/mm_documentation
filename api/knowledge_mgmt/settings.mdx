# /settings

This endpoint allows you to store user-specific information, which currently includes match profiles and custom sets.

## Table of Contents

| Endpoint | Description |
|----------|-------------|
| [GET - All Settings](#get---all-settings) | Retrieve all settings objects. |
| [GET - Settings by ID](#get---settings-by-id) | Retrieve a specific comment by its ID. |
| [POST - Create Settings](#post---create-settings) | Create a new settings object. |
| [PUT - Update Settings Object](#put---update-comment) | Update a preexisting settings object by its ID. |
| [DELETE](#delete---remove-settings) | Delete a specific settings object by its ID. |


## GET - All Settings

### Endpoint
```
GET https://matchmakeredlabs.net/api/settings
```

**Notes:**
- Returns all settings objects if caller has system-wide admin `ReadDescriptor` privilege.
- Otherwise, returns user's own settings object and all organization-specific settings objects for which the user has `ReadDescriptor` privileges.
---

### Request

#### Request Parameters

None required.

### Response

#### Response Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | string | Always "settings_**mainEntityId**" |
| `type` | string | Always "settings" |
| `mainEntityId` | string | The user or organization ID that created the settings object. |
| `matchProfiles` | object array | If it exists, all match profiles owned by the mainEntityId. |
| `customSets` | object array | If it exists, all custom sets owned by the mainEntityId. |
| `_canUpdate` | boolean | True if the user has permission to update the settings object, false if not. |
| `_mainEntityType` | string | Always "user" or "org" |

## POST - Create Settings

### Endpoint
```

```

**Notes:**
- Can be done for a user's own account or for any organization that the user has `WriteDescriptor` privileges on.
